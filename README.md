# LeetCode Clone (Microservices Architecture)


Проект представляет собой микросервисную платформу для решения алгоритмических задач (аналог LeetCode). Система спроектирована с упором на отказоустойчивость и асинхронное взаимодействие.


![](../Desktop/Снимок экрана 2026-03-18 в 19.31.57.png)

## Основные компоненты:

### User Service: 
Управление пользователями, аутентификация (OAuth 2.0 / JWT) и статистика активности. Работает с PostgreSQL для хранения профилей и Kafka/RabbitMQ для уведомлений других сервисов.

### Problem Service: 
Хранилище задач. Интегрировано с PostgreSQL и синхронизируется с системой сабмитов через Kafka.

### Submission Service: 
Ядро системы. Принимает решения пользователей, управляет очередями в RabbitMQ и взаимодействует с внешним движком Judge0 (Self-hosted).

### Notification Service: 
Выделенный сервис для отправки уведомлений пользователям. Использует MongoDB для гибкого хранения истории уведомлений и слушает события из Kafka.

### Технологические особенности:
Event-Driven Design: Использование двух брокеров сообщений (Kafka и RabbitMQ) для разных типов задач (высокая пропускная способность vs гарантированная доставка).
Polyglot Persistence: Комбинирование реляционных баз (PostgreSQL) для транзакционных данных и документоориентированных (MongoDB) для метаданных и уведомлений.
Judge0 Integration: Локальное развертывание движка исполнения кода для обхода ограничений API и повышения безопасности.


# Security & Authentication Flow


Реализована промышленная модель безопасности на базе Spring Security и OAuth2 Resource Server.

![](../Desktop/Снимок экрана 2026-03-18 в 19.49.28.png)

## Многоуровневая защита:

### Client-Level (Basic Auth): 
Защита публичных эндпоинтов. Доступ разрешен только доверенным клиентам по паре clientId:clientSecret через кастомный BasicAuthFilter.
## User-Level (JWT + RSA-256):
### Asymmetric Encryption: 
Токены подписываются приватным RSA-ключом.
### Key Management: 
Реализована ручная сборка ключей через KeyFactory и спецификации PKCS8/X509. Ротация ключей возможна через переменные окружения без пересборки проекта.
## Resource Server & Authorization:
### Stateless Validation: 
Автоматическая проверка подписи и сроков действия JWT.
### Custom JWT Mapping: 
Реализован JwtAuthenticationConverter для преобразования claims токена в GrantedAuthority с поддержкой префиксов ROLE_.
### Security Utilities: 
Разработан SecurityUtil для типизированного извлечения UUID пользователя из SecurityContext.


# Technical Excellence & Performance

![](../Desktop/Снимок экрана 2026-03-18 в 19.53.57.png)


Чтобы система была не только функциональной, но и устойчивой к высоким нагрузкам, реализованы следующие инженерные решения:

## Reliable Messaging (RabbitMQ + DLX):
Асинхронное взаимодействие между сервисами (submission и user) реализовано через брокер RabbitMQ.
### Отказоустойчивость: 
Настроен механизм Dead Letter Exchange (DLX). Если сообщение не может быть обработано (например, из-за временного сбоя потребителя), оно не теряется, а перенаправляется в специальную очередь dlq для последующего анализа и повторной обработки. Это гарантирует консистентность данных в распределенной системе.
## High-Performance Data Access (JdbcTemplate):
В критически важных частях системы (например, user-service) сознательно сделан выбор в пользу JdbcTemplate вместо стандартного Hibernate/JPA.
### SQL Control: 
Прямое управление SQL-запросами позволяет оптимизировать взаимодействие с PostgreSQL, избегая проблемы N+1 и лишних накладных расходов ORM. Это обеспечивает максимальную скорость работы со статистикой пользователей.
## Smart Caching: 
Внедрено многоуровневое кэширование с использованием Spring Cache Abstraction и высокопроизводительного провайдера Caffeine.
### Оптимизация ресурсов: 
Кэшируются наиболее «тяжелые» операции чтения (информация о пользователях и их результативности).
### Eviction Policies: 
Настроены политики вытеснения данных по времени (TTL, TTI) и размеру, что позволяет поддерживать актуальность данных, не перегружая память приложения.
